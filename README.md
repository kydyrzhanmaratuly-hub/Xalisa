# University (статический сайт)

Эта папка содержит статический сайт, готовый для публикации (GitHub Pages).

Файлы в папке `site/`:
- `index.html` — главная страница (кнопка открывает `letter.html`).
- `letter.html` — страница с посланием; кнопка «Отправить» пытается открыть профиль в приложении Instagram и при необходимости переходит на веб-страницу.
- `qrcode.html` — генератор QR-кода для ссылки на ваш сайт (использует CDN `qrcodejs`).

Как опубликовать на GitHub Pages
1. Создайте новый репозиторий на GitHub и запушьте этот проект в него.
2. Вариант A — GitHub Pages с ветки `main`:
   - Перейдите в Settings → Pages и выберите источник (source) — `main` / `root` или ветку `gh-pages`.
3. Вариант B — использовать GitHub Actions (в репозитории уже есть workflow):
   - Создайте Personal Access Token (PAT) и сохраните его в секретах репозитория как `GH_PAT` (требуются права `repo` и `workflow`).
   - Workflow `.github/workflows/deploy.yml` опубликует содержимое папки `site/` в ветку `gh-pages`.

Локальная проверка
- Откройте `site/index.html` в браузере (двойной клик) для тестирования локально.
- Чтобы создать QR-код для опубликованного сайта, откройте `site/qrcode.html`, вставьте URL и нажмите «Сгенерировать».
